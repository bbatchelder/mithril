```tsx
const [contents, { handleNodeClick, handleNodeExpand, handleNodeCollapse }] = useTreeState([
    {
        id: 1,
        label: "Documents",
        icon: "folder-close",
        isExpanded: true,
        childNodes: [
            {
                id: 2,
                label: "Annual Report 2025",
                icon: "document",
                secondaryLabel: <span>4.2 MB</span>,
            },
            {
                id: 3,
                label: "Projects",
                icon: "folder-close",
                isExpanded: true,
                isSelected: true,
            },
        ],
    },
    {
        id: 4,
        label: "Trash",
        icon: "trash",
        disabled: true,
    },
]);

<Tree
    contents={contents}
    onNodeClick={handleNodeClick}
    onNodeExpand={handleNodeExpand}
    onNodeCollapse={handleNodeCollapse}
/>

// Custom controlled usage (full control)
<Tree
    contents={myNodes}
    onNodeClick={(node, path, e) => { /* toggle selection */ }}
    onNodeExpand={(node, path, e) => { /* set isExpanded: true */ }}
    onNodeCollapse={(node, path, e) => { /* set isExpanded: false */ }}
    compact={false}
/>

// Navigate to a node by path
const node = treeNodeFromPath([0, 1], contents);
```

### Tree props

| Prop | Type | Default | Description |
|---|---|---|---|
| `contents` | `TreeNodeInfo[]` | required | Hierarchical node data (controlled). |
| `onNodeClick` | `TreeEventHandler` | — | Called when a node row is clicked (not the caret). |
| `onNodeExpand` | `TreeEventHandler` | — | Called when a collapsed node's caret is clicked. |
| `onNodeCollapse` | `TreeEventHandler` | — | Called when an expanded node's caret is clicked. |
| `compact` | `boolean` | `false` | 24px row height instead of 30px. |
| `className` | `string` | — | Additional class on the outer div. |

### TreeNodeInfo fields

| Field | Type | Default | Description |
|---|---|---|---|
| `id` | `string \| number` | required | Unique identifier. |
| `label` | `ReactNode` | required | Node row text content. |
| `icon` | `IconName` | — | Blueprint icon left of label (after caret). |
| `secondaryLabel` | `ReactNode` | — | Right-aligned content in the node row. |
| `isExpanded` | `boolean` | `false` | Whether children are visible. |
| `isSelected` | `boolean` | `false` | Blue background + white text highlight. |
| `disabled` | `boolean` | `false` | Non-interactive, dimmed. |
| `hasCaret` | `boolean` | auto | Override caret visibility. |
| `childNodes` | `TreeNodeInfo[]` | — | Nested child nodes. |
| `className` | `string` | — | Extra class on the `<li>` wrapper. |

### TreeEventHandler signature

```ts
type TreeEventHandler = (
    node: TreeNodeInfo,
    nodePath: number[],   // index path from root, e.g. [0, 1, 2]
    e: React.MouseEvent<HTMLElement>,
) => void;
```

### useTreeState hook

Convenience hook for simple controlled state. Toggles `isSelected` on click (clears all others)
and `isExpanded` on expand/collapse.

```ts
const [contents, { handleNodeClick, handleNodeExpand, handleNodeCollapse }] = useTreeState(initial);
```

## Design decisions

- **`contents` + controlled state**: Blueprint's class-based `Tree` mutates state externally via
  callback handlers. We follow the same pattern (fully controlled) but provide a `useTreeState`
  convenience hook so simple cases don't require manual immer-style updates.
- **`useTreeState` helper**: Handles the common case (toggle selected on click, toggle expanded on
  caret click) with immutable updates. For complex cases (multi-select, checkbox trees), consumers
  use the raw `onNodeClick`/`onNodeExpand`/`onNodeCollapse` callbacks with their own state.
- **`Collapse` reuse**: Children expand/collapse via the existing `Collapse` component (same
  200ms `ease-bp` animation as Blueprint). `keepChildrenMounted={true}` avoids re-mounting child
  subtrees on re-expand.
- **bp6-class identifiers**: Added `bp6-tree-node-content`, `bp6-tree-node-caret`,
  `bp6-tree-node-caret-none`, and `bp6-tree-node-icon` class names to match Blueprint's DOM
  identifiers, enabling the harness's useEffect-based `data-compare` tagging in the gallery.
- **Depth padding as inline style**: `paddingLeft` is `22 * depth` px — a runtime-computed layout
  value that legitimately needs `style={{ paddingLeft }}`. This is an explicit exception to the
  "no runtime var() in inline styles" rule — it's not a tree-shaken `@theme` token, just arithmetic.
- **No `hasCaret` false-branch on leaf rendering**: Blueprint renders an empty `<Icon>` span for
  undefined icons; analyst-ui skips the span entirely for `icon={undefined}`. This causes the
  "only in analyst: tree-node-icon" informational message but is not a fidelity gap.

## Accepted deltas

`tree-node-icon` "only in analyst" in both themes: Blueprint always renders an icon span wrapper
even when `icon` is undefined; analyst-ui conditionally renders the span. The blueprint-reference
gallery's `icons[1]` querySelector doesn't match because Blueprint's icon count includes those
empty wrappers. Not a visual fidelity issue — the icons are pixel-identical when rendered.
