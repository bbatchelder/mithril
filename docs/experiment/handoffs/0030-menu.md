### data-compare forwarding

`data-compare` on `MenuItem` is forwarded to the inner `<button>` / `<a>` element (not the outer `<li>`).
This mirrors Blueprint's behavior where `htmlProps` spread on the inner anchor. The harness measures
padding, radius, color from the interactive element, not the list item wrapper.

## Design decisions

- **MenuItem renders inner `<button>` by default, `<a>` when `href` is provided**: Blueprint uses
  an `<a>` tag by default (`tagName="a"`). We default to `<button>` for better semantics when no
  href is set. When `href` is provided, we render `<a>` as expected.

- **`data-compare` forwarded to inner element**: The harness measures computed styles on the element
  that has `borderRadius`, `paddingLeft`, etc. For MenuItem, those styles live on the inner
  `<button>`/`<a>`, not the `<li>` wrapper. We extract `data-compare` from liProps at render time
  and pass it to the inner element â€” matching Blueprint's behavior where htmlProps spread onto
  the inner anchor.

- **Icon wrapper has fixed height matching line-height**: `h-[22px]` flex column justifies the icon
  vertically within the line, matching Blueprint's `.bp6-menu-item-icon { height: line-height }`.

- **Blueprint reference: raw `<li>` for divider specimens**: Blueprint's `MenuDivider` component
  does NOT forward arbitrary HTML attributes (`data-compare` etc.) â€” its signature only accepts
  `className, title, titleId`. For gallery specimens we use raw `<li className={Classes.MENU_DIVIDER}>` /
  `<li className={Classes.MENU_HEADER}>` to preserve `data-compare`.

- **`caret-right`, `document`, `trash` icons added**: These icons are needed for the menu gallery
  specimen (submenu caret, document item, delete item). Paths vendored from Blueprint v6.15.

- **Menu renders inline, no portal**: As specified, Menu is just a styled `<ul>`. Popover/ContextMenu
  composition for dropdown behavior is deferred to ContextMenu (Phase 3 #8).

## Accepted Deltas

| Theme | Specimen | Property | Mithril | Blueprint | Why |
|---|---|---|---|---|---|
| Light + Dark | menu-container | height | `223px` | `222px` | 1px total height difference caused by the 1px header height delta (see below). Sub-perceptual. |
| Light + Dark | menu-divider | minWidth | `auto` | `0px` | CSS computed representation difference only. Both are `block` elements with no explicit width â€” browser reports `auto` vs `0px` for the same resolved behavior. Visual output identical. |
| Light + Dark | menu-header | height | `18px` | `17px` | Blueprint's H6 has `line-height: 17px`. Our H6 also uses `leading-[17px]` but browser font rendering rounds to 18px at 14px/semibold. Sub-perceptual 1px difference. |
| Light + Dark | menu-header | minWidth | `auto` | `0px` | Same CSS representation difference as menu-divider. |
