## Decisions made (and why)

### Variant → token mapping

| variant  | font-size | line-height | weight | color (light → dark)          | Blueprint class/mixin |
|----------|-----------|-------------|--------|-------------------------------|-----------------------|
| `body`   | 14px      | 1.28581     | 400    | foreground (dark-gray-1 / light-gray-5) | `base-typography()` |
| `large`  | 16px      | 1.28581     | 400    | foreground                    | `bp6-text-large` |
| `small`  | 12px      | 1.28581     | 400    | foreground                    | `bp6-text-small` |
| `muted`  | 14px      | 1.28581     | 400    | foreground-muted (gray-1 / gray-4) | `bp6-text-muted` |
| `disabled`| 14px    | 1.28581     | 400    | foreground-disabled (gray-1@60% / gray-4@60%) | `bp6-text-disabled` |
| `code`   | 14px      | 1.28581     | 400    | foreground (no color change)  | `monospace-typography()` |
| `h1`     | 36px      | 40px        | 600    | foreground                    | `h1.bp6-heading` |
| `h2`     | 28px      | 32px        | 600    | foreground                    | `h2.bp6-heading` |
| `h3`     | 22px      | 25px        | 600    | foreground                    | `h3.bp6-heading` |
| `h4`     | 18px      | 21px        | 600    | foreground                    | `h4.bp6-heading` |
| `h5`     | 16px      | 19px        | 600    | foreground                    | `h5.bp6-heading` |
| `h6`     | 14px      | 16px        | 600    | foreground                    | `h6.bp6-heading` |

**Heading sizes use explicit pixel values** — Blueprint's `$headings` map (h1: 36px/40px,
h2: 28px/32px, h3: 22px/25px, h4: 18px/21px, h5: 16px/19px, h6: 14px/16px) does NOT
match our token-scale names, so we use `text-[36px]` etc. in CVA.

**Default tagName = `"div"`** — matches Blueprint's `Text` component default (`tagName="div"`).
Heading variants default to their semantic element (h1–h6). `code` defaults to `<code>`.
All can be overridden with `as` or `tagName` props.

**`text-foreground` on base class** — same fix as Icon/Card: sets `color: var(--color-foreground)`
which re-evaluates inside `.dark`, preventing `:root` (light) color from leaking.

**`mb-[12px]` on headings** — Blueprint's `.bp6-heading` sets `margin: 0 0 12px` (3 × $pt-spacing).
Tailwind v4 preflight removes browser default heading margins, so we add them back explicitly.

**`code` variant: no color change** — `monospace-typography()` only sets `font-family: monospace`
and `text-transform: none`. The `bp6-code`/`bp6-code-block` classes (not `bp6-monospace-text`)
set color to `code-foreground`. So `code` variant inherits foreground color, not code-foreground.

**`ellipsize` impl** — uses `useLayoutEffect` + `scrollWidth > clientWidth` on an inner ref
(mirrors Blueprint's `useIsomorphicLayoutEffect`). Sets native `title` attribute to the full
text content when overflow is detected; explicit `title` prop overrides.

**Blueprint reference side** — uses `<div className="bp6-text-*">` for body tiers (NOT the
Blueprint `<Text>` component for most, since those classes are applied via CSS not the component),
and `<h1–6 className="bp6-heading">` for headings. Uses `<Text ellipsize>` for the ellipsize
specimen. `Classes.MONOSPACE_TEXT` → `"bp6-monospace-text"` as a `<div>`.

## Gotchas / things to know

- **`code` variant does NOT set color** — `bp6-monospace-text` is a font-family-only class.
  If you want muted code color, combine `variant="code"` with `className="text-foreground-muted"`.

- **Heading margins removed by Tailwind preflight** — Must explicitly add `mb-[12px]` on
  heading variants. Without it, headings collapse together (no spacing between them).

- **Token-scale names ≠ Blueprint heading sizes** — Our `--text-heading-xl` is 28px, which
  matches h2 (28px). But the naming is coincidental; always reference the `$headings` map in
  `_typography.scss` for authoritative heading pixel values.

- **`text-foreground` base class is essential for dark mode** — Without it, body/large/small/code
  variants would inherit from `body { color: var(--foreground) }` which resolves from `:root`
  (light foreground = #1c2127) even inside `.dark`. The explicit `text-foreground` utility
  re-emits `color: var(--color-foreground)` which re-evaluates per theme.

- **`disabled` color is NOT just `opacity-60`** — it is `rgba(gray-1, 0.6)` in light and
  `rgba(gray-4, 0.6)` in dark. We use `text-foreground-disabled` which maps to the semantic
  `--foreground-disabled` var defined correctly in `:root` and `.dark`.
