## Decisions made (and why)

- **Blueprint is a design spec, not code.** We re-implement on Radix/CVA/Tailwind and port only the tokens.
- **Reference app is isolated** (`tools/blueprint-reference`, React 18) to avoid Tailwind preflight altering
  Blueprint's CSS and to satisfy Blueprint's React-18 peer dep. Keeps `mithril` clean and version-exact.
- **Button color mapping uses literal named utility classes** (`bg-primary`, `bg-blue-3/10`) in
  per-(variant,intent) maps, *not* CSS-var-driven `bg-[var(--x)]`. Reason: Tailwind v4 tree-shakes theme
  vars only referenced via `var()`, so the var-driven approach silently drops palette colors. Named
  utilities are guaranteed to emit. The compoundVariants are generated by mapping over the intent maps to
  stay DRY while keeping the class strings literal (scannable).
- **Warning solid is a special case:** Blueprint fills warning buttons with **orange-5** (`#fbb360`, light
  amber) + black text, not the warning intent base (orange-3). Outlined/minimal warning still use the
  orange-3/2 shades for text/border. Don't "fix" the solid warning to use `--color-warning`.
- **`none` solid button bg = light-gray-5** (`#f7f8f8` in Blueprint) — it nearly matches the page bg; the
  border + `shadow-button` define the button. This is intentional Blueprint behavior.
- **Fresh API:** dropped Blueprint's deprecated props (`large`, `small`, `minimal`, `outlined`,
  `rightIcon`); icons are `ReactNode` (lucide), not icon-name strings.

## Gotchas / things to know

- **Tailwind v4 tree-shaking** (above) — the single most important trap. Use literal utility classes.
- **Flex `align-items: stretch`** corrupts height measurements/visuals in mixed-size rows (small/medium got
  stretched to the large button's 40px). Both galleries set `align-items: center` on the inner row.
- **Reading Blueprint's computed colors:** Blueprint v6.15 emits `oklch()`/`color()` values; convert via a
  1×1 canvas (`ctx.fillStyle = cssColor; getImageData`) to get RGB — naive string parsing fails.
- **Browser permission** is per-origin: :5173 and :5174 each needed a one-time Chrome approval.
